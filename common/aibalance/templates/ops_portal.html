API Usage Guide

1. Create API Key via HTTP API

You can programmatically create API keys using your OPS Key:

cURL Example 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-4", "gpt-3.5-turbo"],
    "traffic_limit": 52428800
  }'
Request Parameters:
allowed_models Array of model names this key can access (required)
traffic_limit Traffic limit in bytes (optional, uses default if not specified)
Response:
{
  "success": true,
  "api_key": "sk-xxxxxxxxxxxxxxxxxxxxxxxx",
  "message": "API key created successfully"
}

2. Use the Created API Key

Use the generated API key with OpenAI-compatible endpoints:

Chat Completion Example 
curl 'http://localhost:8223/v1/chat/completions' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

3. Authentication Headers

X-Ops-Key Your OPS Key for management API calls (create/delete keys)
Authorization: Bearer API Key for AI service calls (chat completions, etc.)
