API Usage Guide

1. Create API Key with Traffic Limit

Create an API key with a specific traffic limit (e.g., 50 MB = 52428800 bytes):

cURL - Create Key with 50MB Limit 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-4", "gpt-3.5-turbo"],
    "traffic_limit": 52428800
  }'

2. Create API Key with Unlimited Traffic

Create an API key without traffic restrictions:

cURL - Create Unlimited Key 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-4", "gpt-3.5-turbo", "claude-*"],
    "unlimited": true
  }'

3. Create API Key with Glob Patterns

Use glob patterns to match multiple models (e.g., gpt-* matches all GPT models):

cURL - Create Key with Glob Patterns 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-*", "claude-*", "memfit-*"],
    "traffic_limit": 104857600
  }'

4. List My API Keys

Get all API keys created by your OPS account:

cURL - List Keys 
curl -X GET 'http://localhost:8223/ops/api/my-keys' \
  -H 'X-Ops-Key: YOUR_OPS_KEY'

5. Update API Key Settings

Update allowed models or traffic settings for an existing API key:

cURL - Update Key 
curl -X POST 'http://localhost:8223/ops/api/update-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "api_key": "mf-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "allowed_models": ["gpt-4", "claude-*"],
    "traffic_limit": 209715200,
    "unlimited": false
  }'

6. Delete API Key

Delete an API key you created:

cURL - Delete Key 
curl -X POST 'http://localhost:8223/ops/api/delete-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "api_key": "mf-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }'

7. Use the Created API Key

Use the generated API key with OpenAI-compatible endpoints:

cURL - Chat Completion 
curl -X POST 'http://localhost:8223/v1/chat/completions' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_GENERATED_API_KEY' \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

Request Parameters Reference

allowed_models Array of model names or glob patterns (required). Examples: ["gpt-4"], ["gpt-*", "claude-*"]
traffic_limit Traffic limit in bytes (optional). Examples: 52428800 (50MB), 104857600 (100MB), 1073741824 (1GB)
unlimited Set to true for unlimited traffic (optional, default: false)

Authentication Headers

X-Ops-Key Your OPS Key for management API calls (create/update/delete keys)
Authorization: Bearer API Key for AI service calls (chat completions, etc.)
