API Usage Guide

1. Create API Key with Traffic Limit

Create an API key with a specific traffic limit (e.g., 50 MB = 52428800 bytes):

cURL - Create Key with 50MB Limit 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-4", "gpt-3.5-turbo"],
    "traffic_limit": 52428800
  }'

2. Create API Key with Unlimited Traffic

Create an API key without traffic restrictions:

cURL - Create Unlimited Key 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-4", "gpt-3.5-turbo", "claude-*"],
    "unlimited": true
  }'

3. Create API Key with Glob Patterns

Use glob patterns to match multiple models (e.g., gpt-* matches all GPT models):

cURL - Create Key with Glob Patterns 
curl -X POST 'http://localhost:8223/ops/api/create-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "allowed_models": ["gpt-*", "claude-*", "memfit-*"],
    "traffic_limit": 104857600
  }'

4. List & Search My API Keys

Get all API keys created by your OPS account. Supports pagination, keyword search and status filtering. You can only ever see keys created by your own OPS account.

cURL - List Keys 
curl -X GET 'http://localhost:8223/ops/api/my-keys?page=1&page_size=20' \
  -H 'X-Ops-Key: YOUR_OPS_KEY'

Search by keyword (matches bound username / remark / api_key) and filter by status. The example below finds active keys whose username, remark or key contains "alice":

cURL - Search & Filter Keys 
curl -G 'http://localhost:8223/ops/api/my-keys' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  --data-urlencode 'q=alice' \
  --data-urlencode 'active=true' \
  --data-urlencode 'page=1' \
  --data-urlencode 'page_size=20'

Query Parameters (List & Search)

q Broad keyword search (optional). Partial match over username, remark and api_key. Special characters are escaped (SQL-injection safe).
username Filter by bound username (optional, partial match).
active Filter by status (optional). true/1 = enabled only, false/0 = disabled only, omit = all.
page Page number (optional, default 1).
page_size Items per page (optional, default 20, max 100).

Response Fields (per key)

api_key / id The API key string and its numeric record ID.
active Whether the key is enabled. A disabled key (false) is rejected on every request.
allowed_models List of model names / glob patterns this key may call.
username / remark / metainfo Bound user info: business username, free-text remark, and JSON metainfo (e.g. OAuth / external binding).
token_used / token_limit Token billing usage and the token limit (the recommended billing dimension).
traffic_used / traffic_limit Byte-traffic usage and limit (legacy dimension).
usage_count / success_count / failure_count Total / successful / failed request counts for this key.
input_bytes / output_bytes / web_search_count Aggregated input / output bytes and web-search invocation count.
last_used_time / created_at / created_by_ops_name Last-used time (omitted if never used), creation time, and the OPS account name that created the key.

5. Update API Key Settings

Update an existing API key. Only the fields you include are changed; omitted fields keep their current values. You can update models, limits, bound user info and status.

cURL - Update Key 
curl -X POST 'http://localhost:8223/ops/api/update-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "api_key": "mf-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "allowed_models": ["gpt-4", "claude-*"],
    "token_limit": 1000000,
    "traffic_limit": 209715200,
    "unlimited": false,
    "username": "alice",
    "remark": "team-a quota",
    "metainfo": "{\"team\":\"a\"}",
    "active": true
  }'

6. Enable / Disable an API Key

Enable or disable a key without deleting it by sending only the active field. Disabling takes effect immediately: a disabled key is removed from the active set and every request using it is rejected. Re-enable it the same way with active: true.

cURL - Disable Key 
curl -X POST 'http://localhost:8223/ops/api/update-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "api_key": "mf-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "active": false
  }'

7. Delete API Key

Delete an API key you created:

cURL - Delete Key 
curl -X POST 'http://localhost:8223/ops/api/delete-api-key' \
  -H 'Content-Type: application/json' \
  -H 'X-Ops-Key: YOUR_OPS_KEY' \
  -d '{
    "api_key": "mf-xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"
  }'

8. Use the Created API Key

Use the generated API key with OpenAI-compatible endpoints:

cURL - Chat Completion 
curl -X POST 'http://localhost:8223/v1/chat/completions' \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer YOUR_GENERATED_API_KEY' \
  -d '{
    "model": "gpt-4",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'

Request Parameters Reference

allowed_models Array of model names or glob patterns (required). Examples: ["gpt-4"], ["gpt-*", "claude-*"]
traffic_limit Traffic limit in bytes (optional). Examples: 52428800 (50MB), 104857600 (100MB), 1073741824 (1GB)
unlimited Set to true for unlimited traffic (optional, default: false)
token_limit Token limit (optional, recommended billing dimension). 0 or omitted means no token limit; set token_unlimited: true to explicitly disable it.
username / remark / metainfo Bound user info (optional). Business username, free-text remark, and JSON metainfo for external/OAuth binding.
active Enable/disable status, update only (optional). true = enabled, false = disabled and immediately unusable. Omit to keep the current status unchanged.

Authentication Headers

X-Ops-Key Your OPS Key for management API calls (create/update/delete keys)
Authorization: Bearer API Key for AI service calls (chat completions, etc.)
