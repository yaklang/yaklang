# 威胁狩猎技能 (Threat Hunting)

以"假设驱动"方法论主动搜寻环境中的威胁，而非被动等待告警。
通过 `grep` 在日志/配置/终端数据中匹配 IOC 与攻击行为特征，结合 `analyze_pcap`
分析网络流量，用 `web_search` 对 IOC 进行情报富化，将狩猎过程与发现写入报告。

与 `incident-response` 的区别：应急响应是**被动响应已发生事件**；威胁狩猎是**主动假设
并验证潜在威胁是否存在**，常用于发现绕过现有检测的高级攻击者。

---

## 1. 假设驱动狩猎方法论

```
提出假设 → 选择数据源与工具 → 验证 → 扩展覆盖 → 记录与反馈检测规则
```

### 1.1 流程

1. **提出假设**：基于威胁情报、ATT&CK 技术、异常迹象，构造可验证假设
   - 例："攻击者可能通过计划任务建立持久化（T1053）"
   - 例："存在某 C2 框架的 beacon 特征流量"
2. **选择数据源与工具**：日志（`grep`）、配置（`read_file`）、流量（`analyze_pcap`）、终端（`system`）
3. **验证**：执行匹配，区分真阳性与误报
4. **扩展覆盖**：发现一条线索后横向扩展（同 IOC、同 TTP、同时间窗）
5. **反馈**：将验证有效的假设沉淀为检测规则（参考 `detection-rule-engineering` 技能）

### 1.2 假设来源

- 威胁情报（新披露的漏洞利用、C2 基础设施、攻击团伙 TTP）
- ATT&CK 技术（针对环境薄弱环节的技术）
- 异常告警的深挖（一个真实告警可能是一系列行为的冰山一角）
- 资产暴露面（面向公网的高危服务）

## 2. IOC 提取与富化

### 2.1 IOC 类型与提取模式

| IOC 类型 | grep 提取模式 | 说明 |
|---------|--------------|------|
| IPv4 | `\b(?:\d{1,3}\.){3}\d{1,3}\b` | 需排除内网/本地地址做外联筛选 |
| 域名 | `\b[a-z0-9]([a-z0-9-]*[a-z0-9])?(\.[a-z]{2,})+\b` | 关注新注册/可疑 TLD |
| URL | `https?://[^\s"'<>]+` | 关注可疑路径与参数 |
| 文件 hash | `\b[a-f0-9]{32}\b|\b[a-f0-9]{64}\b` | MD5 / SHA256 |
| 互操作字符串 | 可执行/样本中的独特字符串 | 需 `read_file` 二进制特征 |

### 2.2 IOC 富化

用 `web_search` 查询已知 IOC 的威胁情报背景：
- 搜索 IP/域名/hash + "threat intel" / "malware" / "report"
- 判断是否为已知 C2、恶意样本、攻击团伙基础设施
- 关联同团伙的其他 IOC，扩展狩猎范围

## 3. MITRE ATT&CK 战术-技术映射与检测模式

> 每项给出 ATT&CK 技术 ID、典型行为特征，以及对应的 grep/检测模式。
> `grep` 模式用于日志/配置/脚本文本；流量类技术配合 `analyze_pcap`。

### 3.1 持久化 (TA0003 Persistence)

| 技术 ID | 技术 | grep 检测模式 |
|---------|------|--------------|
| T1053 | 计划任务 | crontab/systemd/schtask 中的下载执行条目（见 `incident-response` 第 4-5 节） |
| T1547 | 自启动注册表/启动文件夹 | Run 键指向 temp/下载目录的可执行 |
| T1543 | 创建/修改系统服务 | 异常 systemd unit / Windows 服务 |
| T1136 | 创建账户 | 新建账户、UID=0 非根账户、隐藏账户 |
| T1098 | 账户操作 | 新增 SSH authorized_keys、提权操作 |
| T1546 | 事件触发执行（WMI 订阅等） | `__EventFilter` / `CommandLineEventConsumer` |

### 3.2 凭据访问 (TA0006 Credential Access)

| 技术 ID | 技术 | 检测模式 |
|---------|------|---------|
| T1003 | 操作安全账户管理器（LSASS dump / mimikatz） | Sysmon 事件 10（LSASS 访问）；进程名 `mimikatz\|procdump\|lsadump` |
| T1110 | 暴力破解 | 登录日志大量 4625/401；参考 `web-log-analysis` 暴力破解段 |
| T1555 | 凭据存储密码 | 浏览器/密码管理器文件访问异常 |
| T1056 | 输入捕获（键盘记录） | 异常键盘记录器进程、raw input API 调用 |

### 3.3 发现 (TA0007 Discovery)

| 技术 ID | 技术 | 检测模式 |
|---------|------|---------|
| T1046 | 网络服务发现 | 内网扫描痕迹（大量短连接、nmap/masscan UA） |
| T1087 | 账户发现 | `net user`/`getent`/`whoami /groups` 等命令高频出现 |
| T1018 | 远程系统发现 | `nbtstat`/`arp -a`/`ping sweep` 痕迹 |
| T1082 | 系统信息发现 | `uname -a`/`systeminfo` 历史命令 |

### 3.4 横向移动 (TA0008 Lateral Movement)

| 技术 ID | 技术 | 检测模式 |
|---------|------|---------|
| T1021 | 远程服务（RDP/SMB/WinRM/SSH） | 非常规源 IP 的 RDP(3389)/SMB(445)/WinRM(5985)/SSH(22) 登录；4624 类型 10（远程交互） |
| T1077 | Windows 管理共享 | admin$/c$/ipc$ 异常访问 |
| T1550 | 利用替代认证材料（票据传递） | 异常 Kerberos 票据、黄金票据迹象 |
| T1570 | 横向工具转移 | 内网中文件共享传递可执行/脚本 |

### 3.5 命令与控制 (TA0011 Command and Control)

| 技术 ID | 技术 | 检测模式（grep + analyze_pcap） |
|---------|------|-------------------------------|
| T1071 | 应用层协议（HTTP/HTTPS/DNS） | 固定周期 beacon（见 `web-log-analysis` 第 2.8 节）；DNS 隧道（超长子域、TXT 记录异常） |
| T1571 | 非标准端口 | 知名协议跑在非常用端口 |
| T1572 | 协议隧道 | DNS/ICMP/HTTPS 隧道特征流量 |
| T1573 | 加密通道 | 固定 JA3/JA3S 指纹、自签名证书、可疑证书主体 |
| T1090 | 代理（跳板/匿名化） | 连接已知代理/Tor 出口节点 |
| T1105 | 入口工具传输（下载工具） | `wget\|curl\|certutil -urlcache\|bitsadmin` 下载外网载荷 |

### 3.6 防御规避 (TA0005 Defense Evasion)

| 技术 ID | 技术 | 检测模式 |
|---------|------|---------|
| T1070 | 痕迹清除（日志删除） | 日志缺失时段、`rm /var/log/*`、清除事件日志 |
| T1562 | 损害防御（禁用杀软） | 杀软进程异常退出、防护被关闭的事件 |
| T1027 | 混淆文件/信息 | Base64/十六进制编码命令、打包加壳样本 |
| T1036 | 伪装（合法进程名） | svchost.exe 从非系统目录运行、名字混淆（svch0st） |
| T1140 | 解码/解密文件 | `base64 -d`、`certutil -decode` 等解密动作 |

## 4. 流量狩猎指引

使用 `analyze_pcap` 分析抓包文件，关注：

- **异常外联**：内网主机连接陌生外网 IP/域名，尤其是非业务时段
- **C2 beacon**：固定周期、固定大小的请求；Jitter 极小的连接
- **DNS 隧道**：超长子域名、高频 TXT/A 记录查询、知名隧道工具特征
- **数据外传**：异常大的出站流量、非常用端口的大流量传输
- **明文凭据**：HTTP/Telnet/FTP 等明文协议中的敏感信息

参考现有 `pcap/analyze_pcap`、`pcap/decrypt_pcap` 工具（注意后者偏 CTF，生产环境
流量狩猎以 `analyze_pcap` 为主）。

---

## 5. 狩猎扩展策略

发现一条线索后，从三个维度横向扩展：

- **同 IOC 扩展**：该 IP/域名/hash 是否出现在其他主机/时间
- **同 TTP 扩展**：该 ATT&CK 技术是否在其他资产也有痕迹
- **同时间窗扩展**：同一时间段的相关事件（登录、进程、网络）

## 6. 工具使用指南

### 6.1 grep 匹配 IOC 与特征
- 先用宽松模式提取候选（如所有外网 IP），再逐个 `web_search` 富化判断
- 注意排除已知合法地址（CDN、更新服务器）降低误报

### 6.2 analyze_pcap 流量分析
- 指定 pcap 文件路径，关注协议分布、会话列表、异常外联

### 6.3 web_search 情报富化
- 查询 IOC + 威胁情报关键词，判断背景与归属
- 关联同团伙 IOC 扩展狩猎范围

## 7. 狩猎报告格式

每轮狩猎记录：

- **狩猎假设**：陈述假设、依据（情报/ATT&CK/异常）、预期发现
- **覆盖矩阵**：本轮覆盖的 ATT&CK 战术与技术（可用矩阵标记已查/未查）
- **数据源与方法**：使用的日志/配置/流量/工具与具体模式
- **发现**：命中的可疑项（IOC、行为、主机、时间），每项标注
  - 置信度（高/中/低）
  - 是否确认为真阳性
  - 关联的 ATT&CK 技术 ID
- **扩展线索**：由此发现衍生的下一轮狩猎方向
- **检测规则建议**：验证有效的假设，建议沉淀为 Sigma/Yara 规则（参考 `detection-rule-engineering`）
