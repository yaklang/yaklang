## SyntaxFlow 规则输出格式示例

* 使用 `write_rule` 创建新规则
* 使用 `modify_rule` 修改已有规则（指定 modify_start_line 和 modify_end_line）
* 每次生成或修改后，系统会进行 SyntaxFlow 编译验证
* 验证通过后必须使用 `directly_answer` 将完整规则展示给用户

### 正确示例: write_rule（必须包含 desc + 数据流 + alert）

```json
{"@action": "write_rule", "human_readable_thought": "创建漏洞检测规则"}
```

<|GEN_RULE_{{ .Nonce }}|>
desc(
	title: "Test"
	type: audit
	level: info
)
.methodName(* #-> as $source);
alert $source for { message: "检测到风险", level: high };
<|GEN_RULE_END_{{ .Nonce }}|>

### 完成时: directly_answer 展示结果（验证通过后立即执行，禁止再做其他操作）

**有正例（用户提供的漏洞样例=file://、UNSAFE）时**：必须先 require_tool check-syntaxflow-syntax 并传入 path、sample_code、filename、language 进行正例自检，matched=true 后才能 directly_answer。

```json
{"@action": "directly_answer", "human_readable_thought": "规则已生成并通过 check-syntaxflow-syntax（含样例自检）验证，展示完整内容"}
```
（directly_answer 时禁止输出规则代码。系统自动从规则文件读取并展示。你只需输出 1-3 句话的简要总结。验证通过后必须立即 directly_answer。）
