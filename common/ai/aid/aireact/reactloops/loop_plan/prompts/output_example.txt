#### **1. 当你需要为任务规划补充专业或事实性知识时，使用 `search_knowledge`**

**使用场景:**
当用户提出的任务涉及特定行业、专业领域、最新事件或你不确定的专有名词时，直接进行规划可能会导致计划内容空泛、不切实际或错误。此时，应首先使用 `search_knowledge` 来获取背景知识、行业最佳实践、关键数据或成功案例，为后续的规划打下坚实的基础。

**例子如下:**
假设用户的任务是：“帮我为一款新的AI编程助手策划一个市场推广活动。”

```json
{
    "@action": "search_knowledge",
    "input": "面向开发者的AI编程助手市场推广策略"
}
```

#### **2. 当你已经掌握了完成任务所需的信息，并需要将其转化为结构化的执行步骤时，使用 `plan`**

**使用场景:**
在通过 `search_knowledge` 获取了必要信息后，或者当用户任务本身比较清晰、不需要额外知识补充时，使用 `plan` 来将宏观的目标分解为一系列具体、可执行、有先后顺序的子任务。这是将“想法”转变为“行动蓝图”的关键一步。

**例子如下:**
在执行了上一步的 `search_knowledge` 并获得了关于开发者营销的知识（例如：了解到内容营销、技术博客、开源社区合作和开发者大会赞助是有效手段）之后，你可以开始制定具体计划。

```json
{
    "@action": "plan",
    "main_task": "为新款AI编程助手策划一个为期三个月的市场推广活动",
    "main_task_goal": "在活动结束时，实现官网注册用户达到10,000人，并在主流开发者社区（如GitHub, Stack Overflow）中获得至少5篇高质量的正面评测。交付物为一份包含所有活动链接、数据分析和ROI评估的总结报告。",
    "tasks": [
        {
            "task_name": "撰写系列技术博客文章",
            "task_description": "产出三篇高质量的技术博客文章，分别展示AI助手在不同场景下（如代码重构、bug修复、API生成）的核心优势。文章需发布在公司官方博客和至少一个主流开发者媒体上。"
        },
        {
            "task_name": "制作产品演示与教程视频",
            "task_description": "制作一个2分钟的产品核心功能演示视频和一个10分钟的深度使用教程视频。完成视频制作并上传至YouTube和Bilibili等视频平台。"
        },
        {
            "task_name": "发起开源社区合作项目",
            "task_description": "识别3个与产品技术栈相关的热门开源项目，并与项目维护者取得联系，提供免费的Pro账户以换取在项目文档或社区中的推荐。至少达成1个合作。"
        },
        {
            "task_name": "赞助线上开发者大会",
            "task_description": "确定并完成一个线上开发者大会的赞助流程。交付物包括赞助合同的签署、虚拟展台的设计以及一场线上技术分享会的演讲稿（PPT）。"
        },
        {
            "task_name": "建立用户反馈与迭代机制",
            "task_description": "创建一个专门的渠道（如Discord社群或论坛版块）用于收集早期用户的反馈。完成渠道的搭建和宣传，并制定一份每周反馈整理报告的模板。"
        }
    ]
}
```
